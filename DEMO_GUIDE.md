# Clubby — מדריך התקנה לדמו

> מדריך זה מיועד להדגמת Clubby בפני בעלי עסקים. הדמו מציג את שני הצדדים: חוויית הלקוח (אפליקציה) וחוויית בעל העסק (פורטל).

---

## מה תצטרך

| דרישה | פרטים |
|---|---|
| מחשב | Windows 10/11 או macOS |
| Node.js | גרסה 20 ומעלה — [nodejs.org](https://nodejs.org) |
| Docker Desktop | [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/) |
| Supabase CLI | מותקן דרך npm (ראה למטה) |
| דפדפן | Chrome או Edge (מומלץ) |
| טלפון (אופציונלי) | לסריקת QR אמיתית |

---

## שלב 1 — הורדת הפרויקט

```bash
# שכפל את הפרויקט
git clone <REPO_URL> Clubby
cd Clubby
```

---

## שלב 2 — התקנת כלים

### Node.js
הורד והתקן מ-[nodejs.org](https://nodejs.org) (LTS).

### Docker Desktop
הורד מ-[docker.com](https://www.docker.com/products/docker-desktop/) והפעל. **חובה שיהיה רץ לפני הצעד הבא.**

### Supabase CLI
```bash
npm install -g supabase
```

---

## שלב 3 — התקנת תלויות

```bash
# מתוך תיקיית Clubby
npm install
```

---

## שלב 4 — הפעלת מסד הנתונים

```bash
supabase start
supabase db reset
```

> `supabase db reset` מייצר את כל הטבלאות ומזריק נתוני דמו (עסקים, לקוחות, הטבות).

הפקודה תדפיס:

```
API URL:  http://127.0.0.1:54321
Studio:   http://127.0.0.1:54323
DB URL:   postgresql://postgres:postgres@127.0.0.1:54322/postgres
```

---

## שלב 5 — הפעלת האפליקציות

פתח **שני חלונות טרמינל**:

### חלון 1 — אפליקציית לקוח
```bash
cd apps/mobile
npx expo start --web --clear
```
→ פתח [http://localhost:8081](http://localhost:8081)

### חלון 2 — פורטל בעל עסק
```bash
cd apps/portal
npm run dev
```
→ פתח [http://localhost:5174](http://localhost:5174)

## מה להציג בדמו

### צד הלקוח
1. **ארנק** — הצג את שני העסקים, תגי הדרגה (כסף), פיד המבצעים
2. **פיצה רומא** — פרופיל העסק עם תמונת כיסוי, מבצע ברוכים הבאים
3. **שכונתי** — מרקט עם קרדיט ₪20 לרכישה ראשונה
4. **גלה** — עסקים ממוינים לפי מיקום (GPS)
5. **היסטוריה** — מדדי חיסכון

### צד בעל העסק
1. **לוח בקרה** — סטטיסטיקות + גרפים 30 יום
2. **חברים** — רשימת לקוחות + פרטים
3. **כרטיסיות חותמות** — הגדרת כרטיסיית 8 חותמות
4. **פיד פוסטים** — הצג את ההודעות שנראות בארנק הלקוח
5. **קוד QR** — קוד מוכן להדפסה/שיתוף
6. **Platform View** — לוח בקרה מצרפי (לקוח עוגן)

## פתרון בעיות נפוצות

| בעיה | פתרון |
|---|---|
| `supabase start` נכשל | ודא ש-Docker Desktop רץ |
| הדף לא נטען | המתן 30 שניות לאחר `supabase db reset` |
| האפליקציה מציגה "Sign in" ולא הארנק | כנס עם `+972501111111` + OTP `000000` |
| הפורטל ריק | כנס עם `owner@test.com` / `ownerpass123` |
| שגיאת port בשימוש | `supabase stop` ואז `supabase start` מחדש |
